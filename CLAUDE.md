# CLAUDE.md — 料理写真カロリー推定 PoC

このファイルはプロジェクトの「憲法」です。詳細は各規約ドキュメント（`10_management/`）に分割し、ここでは全員（人間・AI）が常に守る最上位ルールと索引だけを置きます。

---

## 1. このプロジェクトは何か

料理写真を見せて「カロリー・料理名・PFC」をAIに推定させ、**OpenAIの最新GPTモデル**と**ai& Inference の激安オープンモデル**を**費用**と**性能**の両面で比較する PoC。主役の実験は「**写真のアングルを積み増すと精度はどこで頭打ちになるか**」。Qiita投稿キャンペーン向け。

最終成果物は **Qiita記事2本**:
1. テーマ本体（カロリー推定モデル比較）
2. **AI駆動開発の方法論**（このプロジェクトの進め方そのものが題材）

→ ②があるため、**「進め方の記録が残ること」も成果物の一部**。ルールは効率だけでなく「あとで記事として見せられるか」で判断する。

企画の全文 → `10_management/背景.md`

## 2. 最重要ルール（例外なく守る）

1. **言語**: ドキュメント・記事・Jira課題・コミットメッセージは**日本語**。コードの識別子・パス・コマンドは英語。
2. **推測の禁止**: モデルID・料金・vision対応可否・APIの挙動は**推測で書かない**。実API（`/models` 等）と公式情報で確定し、確認日と出典を残す。確認前は「要確認」と明記する。
3. **数値の手打ち禁止**: 実験結果の表・グラフ・コスト試算は、必ずスクリプトがログ（JSONL）から生成する。記事・ドキュメントに数値を直接打ち込まない。
4. **秘密情報**: APIキー等は `.env` のみに置き、**コミット厳禁**。`.env.example` にキー名だけ記載。
5. **承認ゲート**: 下記「人間承認が必要な操作」を Claude が独断で実行しない。
6. **トレーサビリティ**: すべての作業は Jira 課題に紐づける。ブランチ名・コミット・設計ドキュメントに課題キー（例 `CAL-12`）を含める。

### 人間承認が必要な操作（実行前に必ず確認を取る）
- **課金が発生するAPIの一括実行**（実験本番の120リクエスト等）← 夜間自律フローの唯一の停止点
- Jira の**削除**・**大量一括変更**（単発の作成・更新・ステータス遷移・コメントは自動でよい）
- 外部公開（Qiitaへの投稿、外部への記事共有）

> 上記以外は自律実行する（起票・設計確定・実装・レビュー・テスト・commit/push/PR/マージ・Jira更新）。人間への質問は必要最低限にする。

## 4. 進め方（2つのフロー・自律実行）

**フロー① Jira起票**:
```
要件 → 課題ドラフト(Fable) → ticket-reviewer でレビュー(Opus) → 修正 → Jira書き込み(Fable)
```

**フロー② 開発**（1チケット＝1サイクル）:
```
要件確認 → 起草:設計/方針を 20_design へ(Opus) → 設計レビュー(Opus) → 実装(Opus)
 → コードレビュー(Opus) → テスト/動作確認(Opus) → 記録(介入ログ) → クローズ(commit/push/PR/マージ/Jira Done)
```

- **モデル振り分け**: Fable=起票・人間への質問／Opus=起草(設計)・レビュー・実装・テスト
- 課金一括実行の直前だけ人間承認。それ以外は自律。
- 詳細 → `10_management/10_開発プロセス規約.md`／自動化 → `10_management/50_自動化ワークフロー規約.md`

## 7. 用語集

- **APE**: Absolute Percent Error（絶対パーセント誤差）。`|推定kcal − 正解kcal| / 正解kcal × 100`。小さいほど良い。
- **PFC**: Protein（タンパク質）/ Fat（脂質）/ Carbohydrate（炭水化物）。
- **S1〜S4**: アングル積み増し実験のステップ（使用画像枚数 1〜4枚）。
- **アングル積み増し実験**: 同じ料理を角度を半分ずつ刻んで撮り、1枚ずつ足して精度の頭打ちを探る本PoCの主役実験。
- **ai& Inference**: OpenAI/Anthropic 互換APIを持つ国内データセンターのAI推論プラットフォーム（比較対象）。
