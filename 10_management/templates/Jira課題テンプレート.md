# Jira課題テンプレート

課題の「説明」欄に貼る定型。AC は合否を機械的に言える粒度で書く。

---

## エピック

```
## 目的 / 勝ち筋
（このエピックで何を達成し、PoCのゴールにどう効くか）

## 含むストーリー
- [ ] CAL-?? ...
- [ ] CAL-?? ...

## 完了条件
（配下ストーリーが全て Done、かつ ○○ が満たされる）
```

---

## ストーリー / タスク

```
## 背景 / 目的
（なぜやるか。1〜2行）

## 受け入れ条件 (AC)
- [ ] （合否を機械的に言える条件1）
- [ ] （条件2）

## Definition of Done (DoD)
- [ ] AC をすべて満たす
- [ ] AIコードレビュー通過・lint/format 通過
- [ ] 動作確認記録あり（40_test/）
- [ ] 関連ドキュメント更新
- [ ] PR マージ済み

## 関連
- ドキュメント: （パス）
- 依存課題: CAL-??
```

---

## サブタスク

```
## やること
（親ストーリーのどの部分か。1行）

## 完了条件
- [ ] （条件）
```

---

## バグ

```
## 事象
（何が起きたか）

## 再現手順
1.

## 期待 / 実際
- 期待:
- 実際:

## 原因 / 対応（判明後）
```
