各開発サイクル⑦で、1チケット＝1段落を追記する。「AIに何をどこまで任せ、人がどこで・なぜ介入したか」を
後から定量・定性で振り返れるようにする（テンプレ: `templates/介入記録テンプレート.md`）。

### CAL-5 共通実装（OpenAI互換クライアント＋JSON固定出力＋JSONLログ）（2026-07-05）
- 任せた範囲: 要件確認→設計起草（`20_design/CAL-5-design.md`）→設計レビュー2回→実装（`30_development/` 一式）→
  コードレビュー2観点→修正→テスト記録→クローズまで、Claude が自律実行。人間承認は課金一括のみ（本サイクルは未発生）。
- 採用率の体感: 設計は初稿の約8割がそのまま。実装コードはほぼ全量 Claude 生成で、レビュー指摘に沿って自己修正。
- 人間の介入: コードには介入なし。人間入力は「料理写真の格納」と「実験品数の方針（1品先行→後で2品）」の
  意思決定のみ。APIキー・正解kcalは未投入で、AC1疎通は追いPRに繰り越し（＝CAL-5 は Done にせずレビュー中で保留）。
- AIレビューで検出:
  - 設計レビュー(design-reviewer)【重大】アングルを S1〜S6 に膨張させた案が憲法の用語集(S1〜S4)・
    120件コスト基準と矛盾 → S1〜S4 既定＋S5/S6 は承認ゲート付き任意拡張に修正して承認。
  - コードレビュー(experiment-validator)【重大】リトライ時に usage/cost が最後の1回で上書きされ、
    安価モデルのコストが系統的に過少計上（中核主張にバイアス） → 全試行 usage を合算し `attempts` を追加、
    リトライを画像なしの整形に変更して解消・approved。
  - 他: 汎用レビューで `extract_json` の後続地の文耐性(raw_decode化)、`confidence` 範囲外での行ごと脱落
    (寛容化)、`image_refs` 絶対パス(相対化)、EXIF向き補正、seed の model 単位opt-in、dry-run ログ隔離 等を検出・修正。
- ハマり / 学び:
  - Workflow(dev-cycle)は未検証かつ自動マージのため、初回かつ一部キー待ちの本チケットは主導実行で各ゲートを回した。
  - 「AIレビューは厚く」が機能: 重大2件はいずれも人間が気づきにくい種類（コスト会計バイアス／憲法整合）で、
    サブエージェントの多観点レビューが実際に価値を出した。記事②の核になる。
  - 数値手打ち禁止の徹底: 単価・為替・正解kcalは全て `要確認` プレースホルダで保持し、確定は CAL-3/CAL-4 に委譲。

### CAL-9/CAL-10 採点・可視化パイプライン（2026-07-05）
- 任せた範囲: CAL-9（採点: APE・料理名正解率・latency・コスト集計）と CAL-10（可視化: 頭打ち折れ線・
  コスト×精度・ランキング・1日コスト試算）を、結合が強いため1設計・1PRで連結実装。設計→設計レビュー2回→
  実装→コードレビュー2観点→修正→テスト→クローズまで自律。キー・正解kcal不要で通すため「合成ログ（mock-logs）＋
  デモ config」の二経路を設計し、end-to-end を実証。
- 採用率の体感: 設計初稿は約7割採用（デモ二経路化・cost母集団・config化で修正）。コードはほぼ全量 Claude 生成。
- 人間の介入: なし（写真も品数方針も前サイクルで確定済み）。ユーザ入力ゼロで1サイクル完走。
- AIレビューで検出:
  - 設計レビュー【重大】デモで頭打ち曲線が描けない（truth/単価が要確認→NaN）／`cost_scenario` の config 欠落／
    `cost_jpy_mean` の母集団未定義 → 二経路化＋config化＋母集団定義で解消し approved。
  - コードレビュー【must-fix】相対 `--config` で `_repo_root` が `.` に潰れ、図が規約の `50_output` でなく
    `30_development/50_output` に出る（実際に踏んでいた）→ `_root` 絶対化＋回帰テストで解消。
  - 【中】`--demo` 付け忘れでデモ値が透かし無しで実出力に混入しうる → demo 性を config 由来（`demo:true`）にして強制整合。
  - 他: 図の色一貫性・空CSVガード・load_labels堅牢化・step数値ソート・nan弾き・CSV文字列往復の統合テストを追加。
- ハマり / 学び:
  - 日本語フォント: `japanize-matplotlib` は Python 3.12 で `distutils` 廃止により import 不可。後継 `matplotlib-fontja`
    に切替え、テスト画像を目視確認してから採用（推測せず実証）。
  - 「デモと実の分離」は透かし・出力先・入力 config の3点が独立だと operator error で崩れる → config フラグで束ねた。
  - dataviz スキルで色覚安全な検証済み配色＋マーカー二次エンコードを適用。記事①の主図（頭打ち曲線）をそのまま流用可能。
  - コードレビューの多観点が「パス解決バグ（人が見落としやすい）」を実測付きで捕捉。記事②の好例。
