---

## このプロジェクトのゴール

1. 料理写真カロリー推定を OpenAI vs ai& Inference で**費用・性能比較**する PoC を完遂する
2. Qiita記事を**2本**出す（①テーマ本体 ②AI駆動開発の方法論）
3. ②のため、**進め方の記録（決定ログ・介入ログ・レビュー履歴）を残しながら**進める

---

## 2つのフロー（自律実行・停止点は課金のみ）

**① Jira起票**
```
要件 → 課題ドラフト(Fable) → ticket-reviewer レビュー(Opus) → 修正 → 書き込み(Fable)
```

**② 開発（1チケット＝1周）**
```
① 要件確認 → ② 起草 20_design(Opus) → ③ 設計レビュー(Opus) → ④ 実装(Opus)
 → ⑤ コードレビュー(Opus) → ⑥ テスト/動作確認(Opus) ─★課金一括のみ承認
 → ⑦ 記録(介入ログ) → ⑧ クローズ(commit/push/PR/マージ/Jira Done ＝自律)
```

- ★ = 人間が止める唯一点（課金一括実行）。それ以外は自律。
- モデル: Fable=起票・質問 ／ Opus=起草・レビュー・実装・テスト。
- 自動連結の仕組み → `50_自動化ワークフロー規約.md`。
