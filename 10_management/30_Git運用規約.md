# 30 Git運用規約

重量級プロセスのため、**課題単位ブランチ＋PR必須**。`main` への直接コミットは禁止。

---

## 1. ブランチ

- `main`: 常にレビュー済み・動く状態を保つ。直接コミット禁止。
- 作業ブランチ: `<type>/<課題キー>-<短い説明>`
  - 例: `feat/CAL-12-base-url-switch`、`docs/CAL-3-experiment-design`、`exp/CAL-20-angle-stacking`
- `<type>`: `feat`（機能）/ `fix`（修正）/ `docs`（文書）/ `exp`（実験）/ `chore`（雑務）

---

## 2. コミット

- 1コミット=1論理変更。こまめに。
- メッセージ形式（**本文は日本語**）:
  ```
  <課題キー>: <要約（命令形・日本語）>

  <必要なら本文。なぜ変更したか>
  ```
  例: `CAL-12: ai&とOpenAIのbase_url切替を実装`
- コミット末尾の Co-Authored-By / Claude-Session 行はハーネス既定に従う。
- **commit・push とも自律**（人間承認は不要。夜間フローの停止点は課金一括実行のみ）。push 前に `git diff --cached` で秘密情報の混入を必ず確認する。

---

## 3. Pull Request

- すべての作業ブランチは PR 経由で `main` に入れる。**PR作成・マージも自律**（トレーサビリティのためPRは残すが、承認待ちにはしない）。
- PRタイトル: `CAL-12: <要約>`
- PR本文に含める:
  - 対応課題（`CAL-12`）へのリンク
  - 変更概要 / AC達成の確認 / 実施したレビューと指摘対応
  - 動作確認内容（`40_test/` の記録への参照）
- レビュー: 先に Claude のサブエージェント／`/code-review` を通し、指摘対応済みにしてから人間承認に出す。

---

## 4. このPoC特有の注意

- **実験ログ（JSONL）・集計結果（CSV）・図表はコミットする**（記事の再現性の根拠）。
- 撮影写真（`90_resources/`）は原則コミット。容量が大きい場合の方針は ADR に残す。
- `.env` 等の秘密情報は絶対にコミットしない（`.gitignore` 済み）。コミット前に `git diff --cached` で混入チェック。
